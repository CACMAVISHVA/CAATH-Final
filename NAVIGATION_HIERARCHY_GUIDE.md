# CAATH Navigation Hierarchy Guide

CAATH navigation follows the enterprise mental model of overview first, execution second, and specialist systems after that.

## Primary Order

1. Dashboard: the enterprise landing surface for executive and operational scanning.
2. Live Workspace: the execution environment for active operator work.
3. Operations: task, compliance, GST, and client work surfaces.
4. Collaboration: team coordination and command-center surfaces.
5. Control: governance and accountability.
6. Knowledge: organizational learning and institutional memory.

## Grouping Principle

Navigation groups should answer one question at a time: Where am I scanning, executing, coordinating, controlling, or learning? Lower-frequency tools remain available, but they should not compete with the primary operating path.

## Future Direction

New modules should be placed by operator intent, not by implementation domain. If a module is mainly for execution, it belongs in Operations. If it frames risk, approval, or audit accountability, it belongs in Control.
