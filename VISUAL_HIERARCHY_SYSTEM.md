# CAATH Visual Hierarchy System

The hierarchy system defines where operator attention should land first, second, and third.

## Attention Order

1. Workflow execution surfaces: live queues, split work rooms, task action zones.
2. Operational status: SLA, risk, throughput, approval readiness.
3. Context panels: knowledge, governance, collaboration, notifications.
4. Navigation and chrome: sidebar, dock rail, tabs, utility controls.

## Typography

- Page titles use medium-large, semibold text with normal letter spacing.
- Section titles use semibold text and compact line height.
- Metadata uses sentence case, smaller size, and muted gray.
- Avoid excessive uppercase. Use it only where compact machine-like labels are essential.

## Color Hierarchy

- White: active headings and critical readable foreground.
- Slate 300-500: body, metadata, inactive navigation.
- Gold: primary action, active execution, scarce emphasis.
- Emerald, red, sky: semantic states only.

## Panel Priority

Hero panels get more spacing, stronger elevation, and clearer heading rhythm. Supporting panels use quieter surfaces, smaller labels, and reduced badge density.
