# CAATH Premium Operational UX Guide

CAATH's premium operational UX keeps enterprise density while removing visual aggression. The interface should feel like an executive operating system: calm surfaces, clear workflow priority, fast scanning, and restrained accents.

## Visual Philosophy

- Use spacing, elevation, and surface tone before borders.
- Reserve gold for primary actions, active execution moments, and selective system emphasis.
- Prefer graphite neutrals over pure black and avoid neon, glow-heavy, or cyber-styled treatments.
- Keep dense operational information, but group it into calm zones with visible hierarchy.
- Make the workflow execution area visually dominant; supporting panels should recede.

## Surface Direction

- Primary surfaces use graphite panels with soft shadow and very low-contrast rings.
- Secondary surfaces use transparent white overlays around `0.03` to `0.07` opacity.
- Borders should be rare and quiet. Use them mostly for tables, separators, and accessibility clarity.
- Nested cards should be replaced with spacing or subdued inner surfaces.

## Action System

- Primary buttons: muted gold background, dark text, minimal shadow.
- Secondary buttons: translucent graphite surface with subtle ring.
- Dangerous actions: muted red surface, not high-chroma red blocks.
- Avoid multiple competing CTAs in one panel. Place the dominant workflow action first.

## Future UX Evolution

Future phases should extend this system through shared tokens, not one-off styling. New operational domains should inherit the same graphite surfaces, restrained semantic color, and workflow-first hierarchy.
