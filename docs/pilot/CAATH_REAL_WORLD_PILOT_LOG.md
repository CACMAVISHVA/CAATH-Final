# CAATH Real World Pilot Log

CAATH is now operating as the daily production system for AVA Associates during internal pilot operations. Use this log to capture real operational friction, repeated confusion, blockers, and improvement opportunities discovered during daily work.

## Logging Rules

- Log issues as soon as they are observed during real use.
- Record the exact module and workflow where the issue appeared.
- Prefer concrete observations over assumptions.
- Do not propose new feature domains here. Capture only what prevents smooth daily operation.
- Move resolved items to `Fixed`, then to `Verified` only after the workflow succeeds in real use.

## Severity Guide

| Severity | Meaning |
| --- | --- |
| P0 | Blocks daily operations |

## Status Guide

| Status | Meaning |
| --- | --- |
| Open | Issue is observed and not yet resolved |
| Fixed | Fix has been implemented or operational workaround exists |
| Verified | User has confirmed the workflow works smoothly in daily use |
