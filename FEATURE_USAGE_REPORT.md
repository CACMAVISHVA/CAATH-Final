# Feature Usage Report

Date opened: 2026-06-04

Purpose: measure which CAATH features provide real operational value during internal pilot usage.

## Usage Tracking Model

Track usage by actual work completed, not feature impressions.

| Metric | Meaning |
|---|---|
| Visit | User opens module |
| Start | User begins a workflow |
| Complete | User finishes workflow inside CAATH |
| Workaround | User leaves CAATH or uses manual process to finish |
| Repeat use | Same user returns to the module on another day |

## Usage-Based Decisions

| Rule | Decision |
|---|---|
| Used daily and supports work completion | Keep and optimize |
| Used weekly and supports audit/management | Keep but reduce visual prominence |
| Rarely used but required for compliance/security | Move to admin/governance area |
| Rarely used and not tied to measurable workflow | Remove or merge |
| Causes repeated confusion | Redesign or consolidate |
