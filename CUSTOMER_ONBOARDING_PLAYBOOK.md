# Customer Onboarding Playbook

Date: 2026-06-04

Purpose: onboard a CA firm into CAATH for pilot operations without hidden technical steps.

## Onboarding Roles

| Role | Responsibility |
|---|---|
| CAATH pilot lead | Own onboarding timeline, expectations, and success criteria |
| Firm owner / Super Admin | Approve firm setup, admin user, and pilot scope |
| Firm admin | Add users, clients, and operational settings |
| Manager | Assign work and run daily operations |
| Staff | Execute tasks, upload documents, and update compliance work |

## Step 1: Create Firm

| Action | Acceptance Criteria |
|---|---|
| Create firm profile | Firm name, address, GSTIN/PAN where relevant, contact, and status are saved |
| Confirm workspace | Firm workspace opens without manual route changes |
| Verify tenant scope | Firm data appears only to authorized firm users |
| Record onboarding owner | Firm has named admin and CAATH support contact |

## Step 2: Create Admin

| Action | Acceptance Criteria |
|---|---|
| Create or invite firm admin | Admin can log in successfully |
| Assign admin role | Admin sees appropriate firm management controls |
| Verify permissions | Admin cannot access God Admin-only controls |
| Confirm logout/login | Admin session resumes correctly |

## Step 3: Invite Users

| Action | Acceptance Criteria |
|---|---|
| Invite manager | Manager can access tasks, clients, compliance, documents, GST workspace |
| Invite staff | Staff can access assigned work and permitted client context |
| Invite optional client user | Client can access only client-appropriate surfaces |
| Verify role boundaries | Each role sees only expected navigation and data |

## Step 4: Add Clients

| Action | Acceptance Criteria |
|---|---|
| Add first client | Client profile persists after refresh |
| Add pilot client set | 10 to 20 clients are ready for pilot workflows |
| Validate client metadata | GSTIN, contact, status, owner, and compliance relevance are correct |
| Link work context | Tasks, compliance, documents, and GST analysis can reference the client |

## Step 5: Start Operations

| Workflow | First Action | Acceptance Criteria |
|---|---|---|
| Dashboard | Manager reviews daily priorities | Top work is visible and actionable |
| Tasks | Create and assign first task | Assignee sees and updates task |
| Compliance | Create or validate first compliance item | Due date, owner, and status persist |
| Documents | Upload first client document | File can be retrieved after refresh |
| GST | Run persisted-data GST pilot analysis | Finding source is clear and not overstated |
| Notifications | Trigger task/compliance event | Notification routes to action |

## First-Day Checklist

| Item | Done |
|---|---|
| Firm created |
| Admin created and logged in |
| Manager and staff invited |
| First client added |
| First task assigned |
| First compliance item created or validated |
| First document uploaded and retrieved |
| GST pilot constraint explained |
| Support channel confirmed |
| Bug and observation process explained |

## Customer Expectation Notes

1. CAATH is ready for controlled pilot operations.
2. GST Intelligence is certified for controlled pilot use with persisted GST operational data.
3. Raw GST upload should not be described as fully certified ingestion.
4. Pilot feedback is expected and will directly shape launch readiness.
5. Any P0 issue pauses customer rollout until resolved.
