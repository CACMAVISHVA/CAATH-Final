# Pilot Deployment Checklist

Date: 2026-06-04

## Before Pilot

- Confirm pilot scope and success criteria.
- Create firm tenant.
- Invite SuperAdmin, Admin, Staff, Client users.
- Resolve Manager role decision.
- Validate RLS and tenant isolation.
- Add first pilot clients.
- Configure GST periods and document categories.
- Validate Client Management CRUD.
- Validate Task lifecycle.
- Validate GST dataset upload and reconciliation.
- Validate Client Portal visibility.
- Validate support ticket creation, feedback, error report, escalation.
- Confirm support SLA and escalation contacts.
- Confirm billing/trial terms.

## During Pilot

- Monitor daily login and workflow completion.
- Review support tickets daily.
- Track failed actions and user confusion.
- Confirm first client onboarding.
- Confirm first GST workflow.
- Confirm first task completion.
- Confirm first document upload/visibility.
- Confirm first notice/compliance workflow.
- Capture user feedback through support tickets.
- Avoid promoting advanced AI/automation unless live workflow value is proven.

## After Pilot

- Review support ticket volume and categories.
- Review workflow completion rates.
- Review unresolved P0/P1 issues.
- Validate customer willingness to pay.
- Confirm plan/billing fit.
- Decide: go paid, extend pilot, or pause.
- Update onboarding playbook with real findings.
- Update production readiness scorecard.

## Pilot Exit Criteria

Pilot can convert to paid only if:

- No open P0 issues.
- Core workflows complete without workaround.
- Tenant isolation validated.
- Customer success can support the firm.
- Firm owner confirms business value.
